git initialize

This is how we begin.


After this point git keeps track of everything you do in its folder.

This step makes the folder into a "repository" or "repo" for short.
git add .
or
git add aDocument.txt
With this command we add any changes we made, to the files, into the repo.
git commit
Is how to save a snapshot of the folder after you have made all the changes you wish to before you continue editing.
With source code this will commit you to making sure your code works.
git status

How do I know what I changed?

This command shows you what branch you are on and what files you have changed since the HEAD commit.

git checkout -branch
This feature allows you to make a branch of the repository allowing you to change code but still have the main, usually stable, version.
git branch
This feature allows you to see what branch you are currently on and what other branches are available to see.
git reset
O.M.G. I just added things that weren't supposed to be in the repository!
This allows you to remove everything from you pending commit so you can only add what you really want to be in there.
git reset --hard HEAD~
Wow I just commited something that was completely wrong.
This allows you to jump back in time. It deletes and resets your entire repository back one whole commit
